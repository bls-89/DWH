База данных “Library” будет состоять из следующих сущностей:

Readers:

Library_card (int) (PK)
surname(varchar)
name(varchar)
patronymic(varchar)
adress(varchar)
phone(int)

Books:
 
id_book (int) (PK)
book_name (varchar)
author_id(int) (FK)
year_of_publish(int)
pages_quantity(int)
cost(int)
number_of-copies(int)
publishing_house_id(int) 


Authors:

author_id(int) (PK)
surname(varchar)
name(varchar)
patronymic(varchar)


Publishing houses:

publishing_house_id(int) (PK) 
name(varchar)
city(varchar)

Book operation:

operation_id(int) (PK)
id_book (int) (FK)
library_card (int) (FK)
operation(bool)
date_of_operation(date)


Связи между сущностями:

Читатели(READERS) и Книги(BOOKS): Многие ко многим. Один читатель может взять несколько книг, и каждую книгу могут взять несколько читателей. Для этого потребуется дополнительная таблица "Операции над книгами"(BOOK OPERATIONS), которая будет содержать информацию о том, какой читатель взял какую книгу.

Книги(BOOKS) и Издательства(PUBLISHING HOUSES): Один ко многим. Одно издательство может выпустить несколько книг, но каждая книга принадлежит только одному издательству.

Книги(BOOKS) и Авторы(AUTHORS): Один ко многим. Один автор может выпустить несколько книг, но каждая книга принадлежит только одному автору.
